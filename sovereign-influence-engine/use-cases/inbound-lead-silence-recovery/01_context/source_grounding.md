# Source Grounding

This bundle is grounded in two key source threads captured from the project:

## Source A — Sovereign Influence Engine definition
Core assertions:
- The SIE is a decision-making infrastructure that owns its own logic, memory, calibration, and authority rules.
- Value is created through a closed loop: ingest → resolve → trust/authority → score/decide → execute → capture outcome → calibrate.
- The system acts through policy-gated interventions and compounds intelligence internally.
- The architecture is explicitly differentiated from prompt-routing tools or ordinary CRMs.

## Source B — Project Memory Candidates
Core assertions:
- The project already contains reusable foundational components:
  - Trust Matrix + Value Matrix
  - Executive Decision Engine
  - Decision Execution Engine
  - First Principles Variable Registry
  - EO System
  - Obsidian/MD standards
  - Portable system prompt contract
- Tier-1 memories should always load.
- Output should be structured, reusable, and machine-ingestible.
- Strategic priority is compounding intelligence and reducing rebuild cost.

## Implication for this bundle
This package does not invent a new system.
It compresses existing doctrine into the first production execution wedge.
